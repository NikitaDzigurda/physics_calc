# Physics Calculator

Калькулятор физических величин с автоматической проверкой размерностей. C++20
ядро + Qt6 Widgets-интерфейс. Поддерживает 7 базовых СИ-размерностей, все
типовые производные единицы (Н, Дж, Па, Вт, В, Ом, …), десятичные префиксы и
основные физические константы. Принимает арифметические выражения,
переменные и неявное умножение.

## Сборка и запуск

Требуется CMake ≥ 3.16, компилятор с C++20 (Apple Clang / GCC 11+ / MSVC 19.3+)
и Qt6 (модуль Widgets).

### macOS (Homebrew)

```bash
brew install qt cmake
cd /path/to/physics_calc
cmake -S . -B build
cmake --build build -j
open build/physics_calc.app
```

Альтернативно бинарь лежит в `build/physics_calc.app/Contents/MacOS/physics_calc`.

CMake сам ищет Qt в типичных Homebrew-префиксах
(`/opt/homebrew/opt/qt`, `/usr/local/opt/qt`). Если Qt установлен в другое
место — задайте `CMAKE_PREFIX_PATH`:

```bash
cmake -S . -B build -DCMAKE_PREFIX_PATH=/path/to/qt6
```

### Linux

```bash
sudo apt install qt6-base-dev cmake build-essential
cmake -S . -B build
cmake --build build -j
./build/physics_calc
```

### Windows (MSVC + Qt)

```bat
cmake -S . -B build -DCMAKE_PREFIX_PATH=C:\Qt\6.7.0\msvc2019_64
cmake --build build --config Release
build\Release\physics_calc.exe
```

## Краткий ввод в работу

После запуска вы видите главное окно с тремя зонами:

```
 ┌─────────────────────────────────────────────────────────┐
 │ >  [ строка ввода выражения ]            [ = ]          │   ← ввод
 ├──────────────────────────────┬──────────────────────────┤
 │ History                      │ Units & Constants        │   ← каталог
 │   > 70 kg * 9.8 m/s^2        │   ▸ SI base              │
 │     = 686 N                  │   ▸ Derived              │
 │   …                          │   ▸ Prefixed             │
 │                              │   ▸ Constants            │
 │                              │   ▸ Math                 │
 │                              ├──────────────────────────┤
 │                              │ Your variables           │   ← переменные
 │                              │  mass   | 70 kg          │
 │                              │  accel  | 9.8 m/s^2      │
 └──────────────────────────────┴──────────────────────────┘
```

Базовый цикл работы:

1. Ввести одно или несколько выражений (по одному на строку).
2. Нажать **Ctrl+Enter** (на macOS — **⌘+Enter**) или кнопку **Run** —
   будут выполнены все непустые строки по порядку. Обычный Enter в
   поле ввода переносит строку, а не запускает вычисление.
3. Строки, начинающиеся с `#` (или текст после `#` в строке), считаются
   комментариями и игнорируются.
4. Результат каждого выражения появляется в панели History со значками:
   `>` — ввод, `=` — результат, `✗` — ошибка, `!` — предупреждение.
5. Присваивания `name = …` создают переменную — она видна в таблице
   «Your variables» и доступна в следующих выражениях. Имя может совпадать
   со встроенной единицей/константой — переменная её затенит на время
   сессии (см. ниже).
6. Двойной клик по символу в каталоге справа вставляет его в текущую
   позицию курсора во вводе.
7. Toolbar:
   - **Clear input** — очистить поле ввода.
   - **Clear history** — очистить лог.
   - **Reset variables** — забыть все пользовательские переменные
     (восстанавливает встроенные значения для затенённых имён).
   - **Help** — встроенная справка.

## Синтаксис выражений

Грамматика (упрощённо):

```
stmt    := IDENT '=' expr | expr
expr    := term     (('+' | '-') term)*
term    := factor   (('*' | '/') factor | implicit-mul factor)*
factor  := unary    ('^' factor)?            // степень — правоассоциативная
unary   := ('+' | '-') unary | atom
atom    := NUMBER | IDENT | '(' expr ')'
```

### Что поддерживается

| Конструкция         | Пример                  | Комментарий                              |
|---------------------|-------------------------|-------------------------------------------|
| Числовые литералы   | `9.8`, `1.6e-19`, `.5`  | Десятичные с опц. экспонентой             |
| Идентификаторы      | `kg`, `m`, `pi`, `mass` | Единицы, константы, ваши переменные       |
| `+ - * /`           | `2 m + 50 cm`           | Стандартная арифметика                    |
| `^` степень         | `m^2`, `s^-2`, `2^10`   | Правоассоциативная                        |
| Скобки              | `(3 V) * (2 A)`         | Группировка                               |
| Унарный плюс/минус  | `-9.8 m/s^2`            |                                           |
| Неявное умножение   | `9.8 m/s^2`, `2 pi`     | Между смежными «атомами»                  |
| Присваивание        | `mass = 70 kg`          | Только в начале строки                    |

### Правила размерностей

- `+`, `-` требуют **совпадения размерностей** операндов
  (`1 m + 1 s` → ошибка).
- `*`, `/` складывают/вычитают показатели размерностей.
- Показатель `^` должен быть скаляром. Для **размерных** величин показатель
  обязан быть целым (`kg^0.5` — ошибка). Для чистых чисел допустимо любое
  вещественное (`2^0.5`).
- Деление на ноль — ошибка времени выполнения.

### Имена переменных

Имя должно начинаться с буквы или `_`, далее буквы/цифры/`_`. Можно
использовать имена, совпадающие со встроенными единицами и константами
(`R`, `V`, `T`, `E`, `m`, …) — ваша переменная **затенит** встроенный
символ до конца сессии. В истории появляется пометка `!` про
затенение. Чтобы вернуть встроенные значения, нажмите
**Reset variables**.

## Примеры запросов

### Базовая арифметика с единицами

```
> 70 kg * 9.8 m/s^2
= 686 N

> (3 V) * (2 A)
= 6 W

> 1 kJ + 500 J
= 1500 J

> 2 m + 50 cm
= 2.5 m
```

### Работа с переменными

```
> mass = 70 kg
mass = 70 kg

> accel = 9.8 m/s^2
accel = 9.8 m/s^2

> force = mass * accel
force = 686 N

> work = force * 10 m
work = 6860 J
```

### Кинематика

```
> v0 = 0 m/s
> a  = 2 m/s^2
> t  = 5 s
> v  = v0 + a*t                       → 10 m/s
> dist = v0*t + a*t^2 / 2             → 25 m
```

### Гравитация (закон всемирного тяготения)

```
> M_earth = 5.972e24 kg
> R_earth = 6371 km
> g_calc  = G * M_earth / R_earth^2   → ≈ 9.82 m/s^2
```

### Электротехника

```
> R = 220 Ohm
> I = 0.5 A
> U = R*I                             → 110 V
> P = U*I                             → 55 W
> E = P * 1 h                         → 198000 J
```

### Газовый закон (молярный)

```
> n = 2 mol
> T = 300 K
> V = 10 cm * 10 cm * 10 cm           → 0.001 m^3
> P = n*R*T / V                       → ≈ 4.99e6 Pa
```

### Релятивистская энергия покоя

```
> E_e = m_e * c^2                     → ≈ 8.187e-14 J
```

### Использование чистых чисел

```
> 2 pi                                → 6.283185307
> 2^10                                → 1024
> (1 + 1/2 + 1/4 + 1/8)               → 1.875
```

## Сообщения об ошибках

Все ошибки печатаются в History со значком `✗`.

| Что произошло                              | Текст ошибки (пример)                                          |
|---------------------------------------------|-----------------------------------------------------------------|
| Несовместимые размерности при `+`/`-`       | `Incompatible dimensions for '+': m vs s`                       |
| Неизвестный идентификатор                   | `Unknown identifier: 'x'`                                       |
| Нецелочисленная степень для размерной       | `Exponent for a dimensioned value must be integer`              |
| Размерный показатель степени                | `Exponent must be dimensionless`                                |
| Деление на ноль                             | `Division by zero`                                              |
| Синтаксическая ошибка                       | `Expected ')' but got '+'`                                      |
| Незавершённое число / неверная экспонента   | `Malformed exponent in number`                                  |

Ошибка не прерывает работу — введите новое выражение, переменные и
история остаются на месте.

## Архитектура проекта

```
physics_calc/
├── CMakeLists.txt
├── README.md
└── src/
    ├── main.cpp                  — точка входа (Qt)
    ├── core/                     — ядро без зависимостей от Qt
    │   ├── Exceptions.h          — CalcError, DimensionMismatch,
    │   │                           ParseError, EvalError, UnknownIdentifier
    │   ├── Dimensions.{h,cpp}    — вектор показателей [M,L,T,I,Θ,N,J]
    │   ├── Quantity.h            — Quantity<T>: число + размерность
    │   ├── Units.{h,cpp}         — UnitRegistry: каталог единиц/констант
    │   ├── Lexer.{h,cpp}         — токенизация строки выражения
    │   ├── Parser.{h,cpp}        — рекурсивный спуск + вычисление
    │   └── Calculator.{h,cpp}    — фасад: env переменных + история
    └── ui/
        └── MainWindow.{h,cpp}    — Qt-окно, виджеты, сигналы/слоты
```

### Слой ядра (`core/`)

**`Dimensions`** — неизменяемая обёртка над `std::array<int, 7>`
с показателями базовых СИ. Операторы `+=`/`-=` соответствуют умножению/
делению величин (показатели складываются/вычитаются). `pow(n)` —
возведение в целую степень. `toString()` форматирует вывод, при точном
совпадении с известной производной единицей выдаёт её символ (`N`, `J`, …).

**`Quantity<T>`** — шаблонный класс «значение + размерность». В арифметике:

- `+`/`-` бросают `DimensionMismatch`, если размерности не совпадают;
- `*`/`/` корректно перемножают значения и складывают/вычитают
  размерности;
- `pow(int)` для возведения; деление на ноль → `EvalError`.

Алиас `Q = Quantity<double>` используется во всём остальном коде.

**`UnitRegistry`** — словарь `name → Quantity` и параллельный список
секций для UI (`SI base`, `Derived`, `Prefixed`, `Constants`, `Math`).
Заполняется в конструкторе на основе базовых СИ.

**`Lexer`** — превращает строку в `std::vector<Token>`: числа, идентификаторы,
операторы `+ - * / ^ ( ) =`. Бросает `ParseError` с позицией символа.

**`Parser`** — рекурсивный спуск по грамматике выше. Дополнительная
особенность: реализует **неявное умножение** в `parseTerm` — если после
очередного `factor` идёт «начало primary» (число, идентификатор или `(`)
без оператора, это интерпретируется как умножение. Возвращает
`Parser::Result` с пометкой, было ли это присваиванием.

**`Calculator`** — единственный объект, который держит состояние:

- словарь пользовательских переменных,
- ссылку на `UnitRegistry`,
- список `HistoryEntry`.

Главный метод `evaluate(string) → HistoryEntry` берёт строку, гоняет её
через лексер и парсер, ловит все исключения семейства `CalcError`,
формирует запись истории (включая ошибки) и возвращает её.

### Слой UI (`ui/MainWindow`)

Чистый Qt Widgets — никакой `.ui` XML, всё собирается кодом в
`buildUi()`. Три ключевых виджета:

- `QLineEdit input_` — ввод выражения, Enter вызывает `onEvaluate()`.
- `QPlainTextEdit history_` — лог `>` запрос / `=` результат / `✗` ошибка.
- Сайдбар: `QTreeWidget reference_` (каталог единиц по секциям) +
  `QTableWidget variables_` (имена и текущие значения переменных).

При клике по строке каталога её символ конкатенируется в строку ввода.
Toolbar содержит действия очистки и встроенный help-диалог.

### Поток данных

```
   QLineEdit.text
        │
        ▼
   Calculator::evaluate(input)
        │
        ├─► Lexer ──► tokens
        │             │
        │             ▼
        │         Parser ──► (использует Calculator::lookup
        │             │       для разрешения идентификаторов
        │             │       — единицы из UnitRegistry или
        │             │       переменные)
        │             ▼
        │         Q result   (или CalcError)
        │
        ▼
   HistoryEntry { input, output, isError }
        │
        ▼
   MainWindow::appendHistory  + refreshVariables
```

### Принципы

- Ядро **полностью независимо от Qt** — можно собрать отдельно и
  использовать в CLI, тестах или другом GUI-каркасе.
- Размерности — `std::array<int,7>` по значению: всё структурное
  сравнение и арифметика тривиальны и без аллокаций.
- Ошибки моделируются исключениями одной иерархии (`CalcError`),
  что упрощает их перехват одним `catch` в `Calculator::evaluate`.

## Как добавить свою единицу или константу

Все элементы каталога регистрируются в одном месте — конструкторе
`UnitRegistry::UnitRegistry()` в `src/core/Units.cpp`. Шаблон:

```cpp
addUnit("Constants",                    // имя секции в UI
        "ly",                           // символ для выражений
        "light year",                   // описание
        scalar(9.4607304725808e15) * m);// значение как Q
```

Сборка:

```bash
cmake --build build -j
```

Новый идентификатор сразу появится в сайдбаре и будет доступен в
выражениях. Если символ уже зарегистрирован, попытка пользователя
переопределить его через `name = …` будет отвергнута с понятным
сообщением.

Для добавления новой **базовой** размерности (например, информационной
энтропии) меняется `Dimensions::kCount`, порядок осей в `enum Axis`,
имена в `base[]` (метод `toString`) и инициализаторы `Array`.
